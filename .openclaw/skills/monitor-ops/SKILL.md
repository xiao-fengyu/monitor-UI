## 项目约束（每次任务强制执行）

1. **先做计划书再干事** — 任何开发任务开始前，必须先输出书面计划书（包含技术选型、步骤分解、验收标准），经用户确认后方可执行。
2. **严格按照制定的计划书执行** — 不得擅自跳过、修改、合并计划中的步骤。如需变更，必须先更新计划书并获得用户同意。
3. **禁止任意重启网关除非得到用户授权** — 任何时候不得自行执行 `openclaw gateway restart` 或任何等效操作。仅在用户明确回复"同意/可以/好"后才能重启。
4. **每次任务后维护项目目录 README.md** — 每次完成一个功能点后，必须更新 `/data/monitor-UI/README.md`，记录本次变更内容、当前进度和下一步计划。
5. **一次只处理一个问题** — 每次对话只聚焦并完成一个具体任务，不得同时推进多个功能点或并行处理多个需求。当前问题完全解决并确认后，再继续下一个问题。

## 执行检查清单

每次任务完成后自检：
- [ ] 本次任务是否事先有计划书？
- [ ] 是否严格按计划书执行？有无擅自变更？
- [ ] 是否未经用户授权重启了网关？
- [ ] README.md 是否已更新？
- [ ] 是否一次只处理了一个问题？有无同时推进多个任务？
